# .instructions.md — Marpex CRM Workspace

## Project Overview

Marpex CRM — lightweight, Apple-style CRM for industrial automation sales.
- **Company**: 20 employees, 5 salespeople, Slovakia
- **Goal**: +30% revenue growth via disciplined pipeline management
- **ERP**: ABRA v26.1 (source of truth for finance); CRM handles sales process
- **Design philosophy**: Apple-style — minimal, intuitive, beautiful, functional

## Key Business Rules

- Every visit MUST produce: a need, an opportunity, or a documented reason why not
- Opportunities require: €value, pipeline stage, next step, deadline
- Pipeline coverage target: ≥3x annual revenue target
- Stagnant opportunity (no activity 30 days) → automatic review flag
- Visit without log entry = visit didn't happen (24h logging window)

## Core Entities

| Entity | Purpose |
|--------|---------|
| **Customers** | Segment, turnover, potential, share of wallet, A/B/C category |
| **Contacts** | Role (decision maker/influencer/user), influence, relationship |
| **Visits** | 11 mandatory fields — the primary data input |
| **Opportunities** | €value, stage (7+2 stages), next step, competition |
| **Pipeline** | Weighted pipeline, Kanban board, coverage metrics |

## Pipeline Stages

1. Identified need (10%) → 2. Qualified (25%) → 3. Technical solution (40%) → 4. Quote delivered (55%) → 5. Negotiation (70%) → 6. Verbally confirmed (90%) → 7. Won (100%) / Lost (0%)

## Coding Conventions

- Language: **TypeScript** (frontend + backend)
- Style: Apple-style UI — max 3 colors, max 3 clicks to any feature
- Performance: <2s page load, <500ms API response
- Mobile-first responsive design
- Offline-first with sync capability
- ABRA integration via REST API (fallback: manual Excel import for MVP)
